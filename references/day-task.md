# 日任务管理

管理每日任务列表，支持随时调整计划、记录完成情况，并根据用户习惯调整任务安排。

## 用户意图识别

| 用户表述 | 意图 |
|---------|------|
| "今天要做什么？" "今天的任务" | 查看今日任务 |
| "今天加上一个会议" | 添加今日任务 |
| "3点的那个会议改到4点" | 修改任务时间 |
| "把写代码任务删了" | 删除任务 |
| "XX做完了" | 标记任务完成 |
| "先做紧急的那个" | 调整任务优先级 |
| "调整一下今天的计划" | 模糊调整，需询问 |

## 接口调用流程

### 1. 查看今日任务

**步骤**：
1. 解析时间上下文获取当前日期
2. 搜索今日任务笔记
3. 获取笔记内容并展示

```bash
# 假设今天是 2024-01-15
ima_api "openapi/note/v1/search_note" '{
  "search_type": 0,
  "query_info": {"title": "2024-01-15 今日任务"},
  "start": 0,
  "end": 1
}'

# 获取内容
ima_api "openapi/note/v1/get_doc_content" '{
  "note_id": "<搜索结果中的note_id>",
  "target_content_format": 0
}'
```

**笔记不存在时的处理**：
- 询问用户："今天还没有任务计划，要创建吗？"
- 或从周/月任务中提取今日相关任务
- 或根据用户习惯建议时段安排

**内容示例**：
```markdown
# 2024-01-15 今日任务

## 上午
- [待开始] 晨会 (9:00-10:00)
- [进行中] 代码审查 (ddl: 1月18日)

## 下午
- [待开始] 项目讨论 (14:00-15:00)
- [待开始] 数据分析 (ddl: 1月22日)

## 晚上
- [待开始] 整理文档

## 已完成
- [已完成] 查收邮件
```

**展示格式**：
```
📅 2024年1月15日 今日任务

🌅 上午
  🔄 进行中：代码审查 (ddl: 1月18日)
  ⏳ 待开始：晨会 (9:00-10:00)

🌤️ 下午
  ⏳ 待开始：项目讨论 (14:00-15:00)
  ⏳ 待开始：数据分析 (ddl: 1月22日)

🌙 晚上
  ⏳ 待开始：整理文档

✅ 已完成
  ✓ 查收邮件
```

### 2. 创建今日任务笔记

**步骤**：
1. 确认日期（默认今天）
2. 检查笔记本是否存在
3. 创建笔记

```bash
# 创建今日任务笔记
ima_api "openapi/note/v1/import_doc" '{
  "content_format": 1,
  "content": "# 2024-01-15 今日任务\n\n## 上午\n- [待开始] 晨会",
  "folder_id": "<task_2024_01的folder_id>"
}'
```

**从周/月任务提取**：
创建时从周任务中提取今日任务：
> "本周计划中今天有以下任务：\n1. 代码审查 (ddl: 1月18日)\n2. 晨会\n\n添加到今日任务吗？"

**基于习惯建议时段**：
参考 `habit_behavior` 中的工作习惯：
> "根据你的习惯，上午是深度工作时段，建议把「代码审查」安排在上午。要这样安排吗？"

### 3. 添加今日任务

**支持多种表述**：

| 用户表述 | 操作 |
|---------|------|
| "今天加上一个会议" | 添加任务，未指定时段则询问或根据习惯建议 |
| "下午3点开会" | 添加任务并指定时间 |
| "把代码审查加到今天上午" | 添加任务并指定时段 |

**示例 - 用户说"今天加上一个会议"**：
```markdown
## 待定时段
+ [待开始] 会议 (待定时段)
```

Agent 可以询问：
> "会议安排在什么时段？根据你的习惯，下午会议较多，要安排在下午吗？"

### 4. 修改任务

**支持的修改**：

| 用户表述 | 操作 |
|---------|------|
| "3点的那个会议改到4点" | 修改任务时间 |
| "把写代码改到上午" | 修改任务时段 |
| "项目报告改成明天做" | 跨日移动任务 |

**示例 - 用户说"3点的那会改到4点"**：
```markdown
# 更新前
## 下午
- [待开始] 项目讨论 (14:00-15:00)

# 更新后
## 下午
- [待开始] 项目讨论 (15:00-16:00)
```

**跨日移动**：
用户说"把代码审查改到明天"时：
1. 从今日任务中移除
2. 询问"是否添加到明天（1月16日）的任务中？"
3. 确认后添加到明日任务

**同步提示**：
"今日任务已更新。是否同步更新到周/月任务中？"

### 5. 删除任务

**步骤**：
1. 确认要删除的任务
2. 更新笔记内容

**用户确认**：
删除前必须确认："确定要删除「会议」任务吗？"

### 6. 标记任务完成

**用户说"XX做完了"**：
```markdown
# 更新前
## 上午
- [进行中] 代码审查

# 更新后
## 已完成
- [已完成] 代码审查
```

**主动记录到习惯数据**：
识别到用户完成了之前未完成过的任务时，主动询问：
> "这个任务你之前没做过，感觉花了多长时间？难度怎么样？"

### 7. 调整任务优先级/顺序

**用户说"先做紧急的那个"**：
- 分析任务 urgency（基于 ddl）
- 重新排序展示

**用户说"调整一下今天的计划"**（模糊表述）：
Agent 询问："您想怎么调整？添加、修改、删除还是重新排序？"

## 时间段划分

### 默认时段

| 时段 | 时间范围 | 用途 |
|------|---------|------|
| 早晨 | 6:00-9:00 | 晨间 routine |
| 上午 | 9:00-12:00 | 深度工作 |
| 中午 | 12:00-14:00 | 午休 |
| 下午 | 14:00-18:00 | 会议、协作 |
| 晚上 | 18:00-22:00 | 个人时间 |
| 深夜 | 22:00-次日6:00 | （通常不安排任务）**

### 基于习惯的个性化时段

从 `habit_behavior` 读取用户的时段偏好：
```markdown
## 工作习惯
- 深度工作时段：上午 9-11 点
- 会议偏好：集中在下午
```

## 基于习惯的任务调整

### 场景1：预估耗时

用户添加任务时，参考 `habit_task` 历史数据：
> "类似任务你之前平均花了 2 小时，这次预估 2 小时可以吗？"

### 场景2：时段建议

参考 `habit_behavior` 建议时段：
> "根据你的习惯，上午是深度工作时段，建议把「代码审查」安排在上午。要这样安排吗？"

### 场景3：习惯洞察

定期总结并提供建议：
> "我发现你最近几次都把重要任务放在上午处理，下午的完成率较低。要不要试着把重要任务都集中在上午？"

**原则**：提供建议，但不强制；最终决策权在用户。

---

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 深夜操作 | 23:00-次日01:00 的表述仍视为"今天" |
| 跨天任务 | 如"加班到凌晨"，可在当天标记，注释说明 |
| 任务过多 | 建议创建周计划细化 |
| 未指定时段 | 根据习惯建议，或放入"待定时段" |

## 与周/月任务的交互

### 从周/月任务同步

创建今日任务时，从周/月任务中提取今日相关任务：
> "本周/本月计划中今天有以下任务：\n1. 代码审查 (ddl: 1月18日)\n2. 晨会\n\n添加到今日任务吗？"

### 向周/月任务同步

今日任务完成后，询问：
> "今日任务「代码审查」已完成，是否同步更新到周/月任务中？"

### 跨日移动

用户把任务改到明天时：
1. 从今日任务移除
2. 询问"是否添加到明天任务？"
3. 如确认，检查明日任务笔记是否存在，不存在则创建
