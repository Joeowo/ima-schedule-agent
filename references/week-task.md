# 周任务管理

管理周任务计划，关注任务在周内哪几天完成。与月任务共用 `plan_{year}_{month}` 笔记本。

## 用户意图识别

| 用户表述 | 意图 |
|---------|------|
| "这周任务好多" "本周很忙" | 触发创建周计划（被动触发） |
| "本周要做什么？" | 查看周任务列表 |
| "这周有什么安排？" | 查看周任务列表 |
| "下周计划一下" | 创建下周计划 |
| "把这个任务加到本周" | 添加周任务 |
| "周一把这个做了" | 添加周任务并指定日期 |

## 接口调用流程

### 1. 周任务创建触发

#### 被动触发

用户主动表达周任务很多时：
```
用户: "这周任务好多"
Agent: "本周任务确实不少，我来帮你创建一个周计划笔记来细化安排吧。"
```

#### 主动提示

Agent 检测到 7 天范围内有 ≥ 8 个任务时：
```
Agent: "本周任务较多（共 9 个），是否创建周计划笔记来细化安排？"
用户: "好"
Agent: [创建周计划笔记]
```

### 2. 查看周任务列表

**步骤**：
1. 解析时间上下文获取当前年月和周次
2. 搜索周任务笔记
3. 获取笔记内容并展示

```bash
# 假设当前是 2024年1月第3周
ima_api "openapi/note/v1/search_note" '{
  "search_type": 0,
  "query_info": {"title": "1月第3周计划"},
  "start": 0,
  "end": 1
}'
```

**笔记不存在时的处理**：
- 询问用户："本周还没有周计划，要创建吗？"
- 或主动提示从月任务中挑选

**内容示例**：
```markdown
# 1月第3周计划

## 周一 1月15日
- [待开始] 代码审查 (ddl: 1月18日)
- [已完成] 晨会

## 周二 1月16日
- [待开始] 代码审查
- [进行中] 数据分析 (ddl: 1月22日)

## 周三 1月17日
- [待开始] 项目讨论

## 周四 1月18日
- [待开始] 代码审查 (ddl: 1月18日) ⚠️ 今天截止

## 周五 1月19日
- [待开始] 提交周报
```

**展示格式**：
```
📅 1月第3周计划 (1月15日 - 1月21日)

📅 周一 1月15日
  ✓ 晨会
  ⏳ 代码审查 (ddl: 1月18日)

📅 周二 1月16日
  🔄 数据分析 (ddl: 1月22日)
  ⏳ 代码审查

📅 周三 1月17日
  ⏳ 项目讨论

📅 周四 1月18日
  ⚠️ 代码审查 (ddl: 1月18日) - 今天截止！

📅 周五 1月19日
  ⏳ 提交周报
```

### 3. 创建周任务笔记

**步骤**：
1. 确认年月和周次（如 2024年1月第3周）
2. 搜索月任务笔记，获取本周相关任务
3. 创建周任务笔记

```bash
# 创建周任务笔记
ima_api "openapi/note/v1/import_doc" '{
  "content_format": 1,
  "content": "# 1月第3周计划\n\n## 周一 1月15日\n- [待开始] 代码审查 (ddl: 1月18日)",
  "folder_id": "<plan_2024_01的folder_id>"
}'
```

**从月任务挑选**：
创建时展示月任务中本周相关的任务：
> "本月计划中有以下任务在本周：\n1. 完成项目报告 (ddl: 1月20日)\n2. 代码审查 (ddl: 1月18日)\n\n添加到周计划中吗？"

### 4. 添加周任务

**支持指定日期**：
- 用户说"周一把这个做了" → 添加到周一
- 用户说"周四之前完成" → 提示具体日期

**示例**：
```markdown
## 周一 1月15日
- [待开始] 新任务 (ddl: 1月18日)
+ [待开始] 另一个任务 (ddl: 1月17日)
```

**向上提醒**：
添加后询问："这个任务是否需要加到月任务中？"

### 5. 更新周任务状态

与月任务相同的状态更新方式：
```markdown
# 更新前
- [待开始] 代码审查 (ddl: 1月18日)

# 更新后
- [已完成] 代码审查 (ddl: 1月18日)
```

**同步提示**：
"周任务已更新。是否同步更新到月任务中？"

## 周次计算

### 周次定义

- 每月第 1 周：包含该月第 1 个周日的周
- 或：包含该月第 1 天的周

**示例**：
```
2024年1月：
- 第1周：1月1日 - 1月7日
- 第2周：1月8日 - 1月14日
- 第3周：1月15日 - 1月21日
- 第4周：1月22日 - 1月28日
- 第5周：1月29日 - 1月31日
```

### 当前周次计算

基于平台提供的当前日期：
1. 获取当前月份和日期
2. 计算属于该月的第几周
3. 搜索对应的周任务笔记

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 跨月周 | 如1月第5周跨到2月，笔记仍在1月笔记本，标题注明跨月 |
| 笔记不存在 | 询问"本周还没有周计划，要创建吗？" |
| 周次计算错误 | 优先响应用户明确说的"本周"、"下周" |
| 任务在多日重复 | 如"每天晨会"，可在每日都标注 |

## 与月/日任务的交互

### 从月任务同步

创建周计划时，从月任务中挑选本周相关的任务：
> "本月计划中有以下任务在本周：\n1. 完成项目报告 (ddl: 1月20日)\n2. 代码审查 (ddl: 1月18日)\n\n添加到周计划中吗？"

### 向日任务分发

用户说"安排今天的任务"时，从周任务中提取今日任务：
> "本周计划中今天（1月15日）有以下任务：\n1. 代码审查 (ddl: 1月18日)\n2. 晨会\n\n要添加到今日任务吗？"
