# 月任务管理

管理月度任务计划，关注任务 ddl 和生命周期。

## 用户意图识别

| 用户表述 | 意图 |
|---------|------|
| "这个月有哪些任务？" | 查看月任务列表 |
| "本月计划是什么？" | 查看月任务列表 |
| "添加一个任务到本月计划" | 添加月任务 |
| "把XX任务加到下个月" | 添加月任务 |
| "XX任务做完了" | 更新任务状态 |
| "取消XX任务" | 删除任务 |
| "修改XX任务的ddl" | 修改任务 |

## 接口调用流程

### 1. 查看月任务列表

**步骤**：
1. 解析时间上下文获取当前年月
2. 搜索月任务笔记
3. 获取笔记内容并展示

```bash
# 假设当前是 2024年1月
# 1. 搜索月任务笔记
ima_api "openapi/note/v1/search_note" '{
  "search_type": 0,
  "query_info": {"title": "2024年1月计划"},
  "start": 0,
  "end": 1
}'

# 2. 获取笔记内容
ima_api "openapi/note/v1/get_doc_content" '{
  "note_id": "<从搜索结果获取>",
  "target_content_format": 0
}'
```

**笔记不存在时的处理**：
- 询问用户："本月还没有任务计划，要创建吗？"
- 确认后调用 `import_doc` 创建新笔记

**内容示例**：
```markdown
# 2024年1月计划

## 待开始
- [待开始] 完成项目报告 (ddl: 1月20日)
- [待开始] 代码审查 (ddl: 1月18日)

## 进行中
- [进行中] 数据分析 (ddl: 1月22日)

## 已完成
- [已完成] 提交周报 (ddl: 1月15日)

## 已延期
- [已延期] 用户调研 (ddl: 1月10日 → 1月25日)
```

**展示格式**：
```
📅 2024年1月计划

📋 待开始 (2)
  1. 完成项目报告 (ddl: 1月20日)
  2. 代码审查 (ddl: 1月18日)

🔄 进行中 (1)
  1. 数据分析 (ddl: 1月22日)

✅ 已完成 (1)
  1. 提交周报 (ddl: 1月15日)

⏰ 已延期 (1)
  1. 用户调研 (1月10日 → 1月25日)
```

### 2. 添加月任务

**步骤**：
1. 确认年月（用户明确指定或使用当前月）
2. 检查月任务笔记是否存在
3. 不存在则创建，存在则追加

```bash
# 创建新笔记
ima_api "openapi/note/v1/import_doc" '{
  "content_format": 1,
  "content": "# 2024年1月计划\n\n## 待开始\n- [待开始] 新任务 (ddl: 1月25日)",
  "folder_id": "<plan_2024_01的folder_id>"
}'

# 或追加到已有笔记
ima_api "openapi/note/v1/append_doc" '{
  "note_id": "<月任务笔记ID>",
  "content_format": 1,
  "content": "\n- [待开始] 新任务 (ddl: 1月25日)"
}'
```

**向上提醒**：
添加后询问："这个任务是否需要加到本周/今日计划中？"

### 3. 更新任务状态

**步骤**：
1. 获取笔记内容
2. 解析并更新对应任务的状态
3. 追加更新后的内容

**用户说"XX做完了"**：
```markdown
# 更新前
- [待开始] 完成项目报告 (ddl: 1月20日)

# 更新后
- [已完成] 完成项目报告 (ddl: 1月20日)
```

**实现方式**：
- 获取内容后解析任务列表
- 找到匹配的任务，更新状态
- 可以用 `append_doc` 追加更新后的任务，或告知用户新状态

### 4. 修改任务

**支持的修改**：
- 修改 ddl
- 修改任务描述
- 标记延期

**示例 - 用户说"把项目报告延期到25号"**：
```markdown
# 更新前
- [待开始] 完成项目报告 (ddl: 1月20日)

# 更新后
- [已延期] 完成项目报告 (ddl: 1月20日 → 1月25日)
```

**同步提示**：
"月任务已更新。是否同步更新到周/日任务中？"

### 5. 删除任务

**步骤**：
1. 确认要删除的任务
2. 获取笔记内容
3. 移除对应任务行
4. 更新笔记内容

**用户确认**：
删除前必须确认："确定要删除「项目报告」任务吗？"

---

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 笔记不存在 | 询问"本月还没有任务计划，要创建吗？" |
| 跨月操作 | 用户说"下个月"时，解析为下个月 |
| 任务名称模糊 | 列出候选任务让用户选择 |
| ddl 已过期 | 询问"这个任务的ddl已过，是要延期还是标记完成？" |

## 与周/日任务的交互

### 创建周任务时的提示

当检测到 7 天范围内有 ≥ 8 个任务时，主动询问：
> "本周任务较多（共 N 个），是否创建周计划笔记来细化安排？"

### 从月任务同步到周/日

创建周/日计划时，展示月任务中本周/今日相关的任务：
> "本月计划中有以下任务在本周：\n1. 完成项目报告 (ddl: 1月20日)\n2. 代码审查 (ddl: 1月18日)\n\n是否添加到周计划中？"
