# 习惯追踪

记录用户任务执行数据（`habit_task`）和工作习惯/性格（`habit_behavior`），用于优化任务安排和提供个性化建议。

## 用户意图识别

| 用户表述 | 意图 |
|---------|------|
| "记录一下这个任务花了多久" | 记录任务执行数据 |
| "我写代码一般要多久" | 查看历史任务数据 |
| "我的工作习惯是什么" | 查看习惯记录 |
| "我发现我早上效率高" | 用户主动告知习惯 |
| "看看我的习惯数据" | 查看所有习惯记录 |

## 接口调用流程

### 1. 查看习惯记录

#### habit_task（任务执行记录）

**步骤**：
1. 搜索 `habit_task` 笔记
2. 获取笔记内容并展示

```bash
ima_api "openapi/note/v1/search_note" '{
  "search_type": 0,
  "query_info": {"title": "habit_task"},
  "start": 0,
  "end": 1
}'
```

**内容示例**：
```markdown
# habit_task

## 任务执行记录

| 任务类型 | 预估时间 | 实际时间 | 完成时段 | 质量自评 | 日期 |
|---------|---------|---------|---------|---------|------|
| 写代码 | 2h | 3h | 下午 | 良好 | 1月15日 |
| 会议 | 1h | 1h | 上午 | 一般 | 1月15日 |
| 写文档 | 3h | 4h | 上午 | 优秀 | 1月14日 |
| 代码审查 | 1h | 1.5h | 下午 | 良好 | 1月13日 |
```

**展示格式**：
```
📊 任务执行记录

最近记录：

1. 写代码 | 预估2h → 实际3h | 下午 | 良好 | 1月15日
2. 会议 | 预估1h → 实际1h | 上午 | 一般 | 1月15日
3. 写文档 | 预估3h → 实际4h | 上午 | 优秀 | 1月14日

💡 洞察：你写代码的实际时间比预估多50%，建议预估时留更多缓冲。
```

#### habit_behavior（工作习惯和性格）

**步骤**：
1. 搜索 `habit_behavior` 笔记
2. 获取笔记内容并展示

```bash
ima_api "openapi/note/v1/search_note" '{
  "search_type": 0,
  "query_info": {"title": "habit_behavior"},
  "start": 0,
  "end": 1
}'
```

**内容示例**：
```markdown
# habit_behavior

## 工作习惯
- 深度工作时段：上午 9-11 点
- 会议偏好：集中在下午
- 任务切换频率：每 2 小时切换一次
- 午休习惯：13:00-14:00

## 性格特点
- 对 ddl 敏感：会在 ddl 前集中发力
- 完美主义：对核心任务追求高质量
- 喜欢提前规划：通常提前一周安排任务

## 时间习惯
- 深夜边界：23:00-01:00 仍视为"今天"
- 周一默认："本周"指当前周
```

**展示格式**：
```
🧠 工作习惯

⏰ 深度工作时段：上午 9-11 点
📅 会议偏好：集中在下午
🔄 任务切换频率：每 2 小时切换一次
😴 午休习惯：13:00-14:00

💡 性格特点
- 对 ddl 敏感：会在 ddl 前集中发力
- 完美主义：对核心任务追求高质量
- 喜欢提前规划：通常提前一周安排任务

🕐 时间习惯
- 深夜边界：23:00-01:00 仍视为"今天"
```

### 2. 记录任务执行数据

#### 用户主动记录

**用户说"记录一下这个任务花了多久"**：

步骤：
1. 询问任务详情（预估时间、实际时间、完成时段、质量自评）
2. 获取 `habit_task` 笔记内容
3. 追加新记录

```bash
# 追加到 habit_task
ima_api "openapi/note/v1/append_doc" '{
  "note_id": "<habit_task笔记ID>",
  "content_format": 1,
  "content": "| 写代码 | 2h | 3h | 下午 | 良好 | 1月15日 |\n"
}'
```

**询问模板**：
```
请提供以下信息：
1. 任务类型/名称：？
2. 预估时间：？
3. 实际时间：？
4. 完成时段（上午/下午/晚上）：？
5. 质量自评（优秀/良好/一般/待改进）：？
```

#### Agent 主动记录

**触发时机**：识别到用户完成了之前未完成过的任务时

**示例**：
```
用户: "代码审查做完了"
Agent: "这个任务你之前没做过，感觉花了多长时间？难度怎么样？"
用户: "大概1.5小时，不算太难"
Agent: [记录到 habit_task]
```

### 3. 更新习惯记录

#### 用户主动告知

**用户说"我发现我早上效率高"**：

步骤：
1. 获取 `habit_behavior` 笔记内容
2. 追加或更新对应习惯条目

```bash
# 追加到 habit_behavior
ima_api "openapi/note/v1/append_doc" '{
  "note_id": "<habit_behavior笔记ID>",
  "content_format": 1,
  "content": "- 深度工作时段：上午效率最高\n"
}'
```

#### Agent 观察并总结

**触发时机**：观察到模式后

**示例**：
```
Agent: "我发现你最近几次都把重要任务放在上午处理，这是你偏好的方式吗？"
用户: "对，上午我精神最好"
Agent: [记录到 habit_behavior]
```

#### 周总结时更新

在编写周总结日记时，回顾本周行为模式，更新 `habit_behavior`：

**示例**：
```
Agent: "本周回顾：你在上午完成的任务质量普遍更高，下午容易分心。要更新到习惯记录吗？"
```

### 4. 创建习惯笔记

**笔记不存在时的处理**：

#### habit_task

```bash
ima_api "openapi/note/v1/import_doc" '{
  "content_format": 1,
  "content": "# habit_task\n\n## 任务执行记录\n\n| 任务类型 | 预估时间 | 实际时间 | 完成时段 | 质量自评 | 日期 |\n|---------|---------|---------|---------|---------|------|\n| 写代码 | 2h | 3h | 下午 | 良好 | 1月15日 |",
  "folder_id": "<habit笔记本的folder_id>"
}'
```

#### habit_behavior

```bash
ima_api "openapi/note/v1/import_doc" '{
  "content_format": 1,
  "content": "# habit_behavior\n\n## 工作习惯\n- 深度工作时段：上午 9-11 点\n\n## 性格特点\n- 对 ddl 敏感：会在 ddl 前集中发力",
  "folder_id": "<habit笔记本的folder_id>"
}'
```

## 基于习惯的任务建议

### 场景1：预估耗时

用户添加任务时，参考 `habit_task` 历史数据：

**示例**：
```
用户: "今天加一个写代码的任务"
Agent: "根据记录，你写代码平均耗时 2.5 小时，这次预估 2.5 小时可以吗？"
```

### 场景2：时段建议

参考 `habit_behavior` 建议时段：

**示例**：
```
用户: "安排今天的代码审查"
Agent: "根据你的习惯，上午是深度工作时段，建议把「代码审查」安排在上午。要这样安排吗？"
```

### 场景3：习惯洞察

定期总结并提供建议：

**示例**：
```
Agent: "💡 习惯洞察：
- 你写代码的实际时间比预估平均多 50%，建议预估时留更多缓冲
- 下午的会议完成率较高，可以继续集中安排会议
- 深夜（23点后）你几乎不完成任务，建议避免在这个时段安排重要任务"
```

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 笔记不存在 | 创建新笔记，初始化表格结构 |
| 数据过多 | 展示最近 10-20 条记录，完整数据可按需获取 |
| 习惯冲突 | 用户主动告知与观察到的模式冲突时，以用户说的为准 |
| 首次记录 | 引导用户填写基础信息，后续逐步完善 |

## 数据结构

### habit_task 推荐字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 任务类型 | 任务分类 | 写代码、会议、写文档 |
| 预估时间 | 事前预估 | 2h、1.5h |
| 实际时间 | 实际花费 | 3h、1h |
| 完成时段 | 上午/下午/晚上 | 下午 |
| 质量自评 | 优秀/良好/一般/待改进 | 良好 |
| 日期 | 完成日期 | 1月15日 |

### habit_behavior 推荐分类

| 分类 | 示例条目 |
|------|---------|
| 工作习惯 | 深度工作时段、会议偏好、任务切换频率 |
| 性格特点 | 对 ddl 敏感、完美主义、规划偏好 |
| 时间习惯 | 深夜边界、周次指代 |
| 注意力模式 | 深度工作时长、分心频率 |
